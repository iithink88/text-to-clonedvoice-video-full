# 新手安装指导（MiniMax 克隆声准备）

> 第一次跑这个 skill 之前，先把 MiniMax 账号和"你的声音"准备好。
> ⚠️ **重点纠正**：克隆声音是在**本地用 API 请求**完成的（上传音频 → 拿 voice_id），不是去 MiniMax 控制台点"音色复刻"按钮。整个过程钱花得极少（见末尾价格速览）。

---

## 第 1 步：打开 MiniMax 开放平台，注册登录

1. 打开官网：<https://platform.minimaxi.com>
2. 用手机号注册并登录（国内平台，不用翻墙）。
3. 登录后进入「控制台」。

---

## 第 2 步：充值余额

1. 控制台右上角 → 「用户中心」→「充值」（或直接访问 <https://platform.minimaxi.com/user-center/payment/balance>）。
2. 选「按量计费」模式，充值余额。
3. **充多少？** 建议首次充 **10~20 元** 就够用很久了——克隆一个声音 + 几十条视频的配音都花不完（见第 5 步算账）。最低充值额度以平台页面为准。

> ⚠️ 强调：这是「先用后扣」，不是包月。不做视频不扣钱。

---

## 第 3 步：获取 API Key（GroupId 可选，别被它卡住）

1. 控制台 → 「接口密钥」（或「API 密钥管理」）。
2. 点「创建密钥」，复制生成的 Key（一长串，形如 `sk-xxxx...`）。
3. **关于 GroupId（重要）**：本 skill 的 `generate_clone_voice.py` 里，`GroupId` 是**可选参数**——配置里 `group_id` 留空时，请求根本不带这个参数（你现在这份配置 `group_id` 就是空的，能正常跑）。**所以新手不用专门去拿 GroupId**，把它留空就行；只有遇到接口明确报错"需要 GroupId"时，再去控制台地址栏 URL 里复制 `group_id=xxxx` 填进配置。

> 关于 Key 类型：
> - 「按量计费 / 余额」模式 → 填配置文件里的 `balance_api_key` 字段，运行时加 `--billing balance`。
> - 「Token Plan 订阅套餐」→ 填 `token_plan_api_key` 字段（脚本默认走这个）。
> - 嫌麻烦：**新手直接用「按量计费 + 余额」最直观**，对应 `balance_api_key`。

---

## 第 4 步：本地克隆你的声音（核心步骤，不是在平台上克隆！）

skill 不会自动帮你"生"出一个声音，你得在**本地**用 MiniMax 的接口把你的声音样本克隆出来。流程是：

```
voice_sample.mp3  →  /v1/files/upload  →  file_id  →  /v1/voice_clone  →  voice_id
```

### 4.1 准备声音样本

录一段 **10 秒 ~ 5 分钟、清晰、无背景音乐** 的你自己说话的音频，存为 `voice_sample.mp3`（手机录就行，越干净还原度越高）。

### 4.2 上传音频，拿到 file_id

用 curl（本地执行）：

```bash
curl -X POST "https://api.minimax.chat/v1/files/upload" \
  -H "Authorization: Bearer 你的API_KEY" \
  -F "file=@voice_sample.mp3"
```

返回里找 `file.file_id`（或类似字段），记下来。

### 4.3 调用声音克隆接口，拿到 voice_id

```bash
curl -X POST "https://api.minimax.chat/v1/voice_clone" \
  -H "Authorization: Bearer 你的API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "上一步拿到的file_id",
    "voice_name": "我的克隆声"
  }'
```

返回里找 `voice_id`（形如 `MyVoice20260528` 这种一串字符），复制下来。

> 💡 这一步就是"本地克隆"：你在本机发请求完成音色克隆，**不是**去控制台点"音色复刻"。接口参数以当前 MiniMax 文档为准，如果字段名有出入，以控制台「语音 → 音色复刻」的 API 文档说明为准。
>
> 💰 **这一步要钱，但只收一次**：克隆一个音色 **9.9 元/个**，首次用这个音色去合成音频时才扣费（单纯发起克隆请求不立即扣）。克隆好之后，这个声音你永久能用，后面每条视频配音都不用再克隆。

---

## 第 5 步：把凭据写进配置文件

在本地创建（或编辑）配置文件，路径随意，建议放：

```
~/.codex/skills/jiajia-digital-human-video/private/config.json
```

（skill 默认就找这个路径；也可以放别处，运行时用 `--config 你的路径` 指定。）

文件内容示例：

```json
{
  "minimax": {
    "group_id": "",
    "voice_id": "填第4步拿到的voice_id",
    "token_plan_api_key": "如果用TokenPlan就填这",
    "balance_api_key": "如果用按量计费余额就填这",
    "preferred_model": "speech-2.8-turbo"
  }
}
```

填完后跑 TTS（合成配音）时：

```bash
# 按量计费（余额）模式
python3 scripts/generate_clone_voice.py --config 你的config.json --text 文案.txt --billing balance

# 或 Token Plan 模式（脚本默认）
python3 scripts/generate_clone_voice.py --config 你的config.json --text 文案.txt
```

- 一条 300 字口播文案 = 600 字符 ≈ 0.12 元音频费
- 第一次做视频：9.9（克隆声音，仅首次）+ 0.12（配音）≈ **10 元**，得到一个永久音色 + 第一条视频配音
- 之后每做一条新视频：音频费才 **几毛钱**

> 所以"不用自己配音、AI 克隆你的声音、给文案就出片"，成本基本可以忽略。

## 常见问题

**Q：一定要拿 GroupId 吗？**
A：不用。config 里 `group_id` 留空，请求就不带它，能正常跑（当前配置就是这么设的）。只有接口明确报错要 GroupId 才去补。

**Q：声音克隆出来不像我？**
A：样本音频要清晰、无背景音乐、本人正常说话 10 秒以上。嘈杂或带 BGM 会拉低还原度。

**Q：能换声音吗？**
A：能。再克隆一个新音色（又 9.9 元），把 `voice_id` 改成新的即可。

**Q：克隆必须在平台上点按钮吗？**
A：不是。克隆是通过本地调 MiniMax 的 `voice_clone` 接口完成的（见第 4 步），控制台里的"音色复刻"只是同一个能力的网页入口，二选一即可。

**Q：不想用 MiniMax 了？**
A：换任何兼容 T2A v2 接口的 TTS 都行，改 `generate_clone_voice.py` 里的请求即可；本 skill 的画面生成部分不依赖具体 TTS 厂商。
